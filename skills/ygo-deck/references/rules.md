# 卡组构筑规则

## 主卡组
- Main: 40-60张, 推荐40-44(40张最稳定抽核心)
- <40张: 非法卡组, 无法投入对战
- >60张: 非法卡组, 无法投入对战
- 40-44张: 推荐范围, 抽核心概率最高
- 45-50张: 可用但需理解为什么(特定战术需求)
- 51-60张: 不推荐, 抽核心概率显著降低

## 额外卡组
- Extra: 0-15张
- 0张: 纯主卡组构筑, 确认是有意为之(如某些黄金国/纯手坑控制)
- 1-11张: 警告, 可能缺少资源, 确认理由
- 12-15张: 推荐范围, 资源利用充分
- >15张: 非法, 超过规则上限

## 同名卡限制
- 同名卡≤3张(按卡名判定, 不是按id)
- alias=0: 正常卡, 用自己的id计数
- alias≠0且与id差≤10: 异画卡, 与alias的卡视为同名
- alias≠0且与id差>10: 同名不同效果(如白龙之落胤→阿不思), 与alias共享≤3限制
- 计数方法: 将所有卡的alias解析后, 同一alias key下总数量≤3

## lim列(禁限卡表)规则
- lim0: 禁止(带0张) — 例: 水晶机巧-继承玻纤, 神弓, 捕食植物蛇颈龙
- lim1: 限制1(≤1张) — 例: 增殖G, 墓指, 羽毛扫, 割草
- lim2: 限制2(≤2张) — 例: 灰流丽, 神之宣告
- 空/无lim: 无限制(≤3张)
- lim检查优先级: 先查alias的lim, 再查自身id的lim
- 构筑时必须遵守lim限制, 否则非法

## 卡片类型判定
- 额外怪兽判定: type & 0x4802040 > 0 (融合0x40 | 同调0x2000 | 超量0x800000 | 连接0x4000000)
- 怪兽: type & 0x1 > 0
- 魔法: type & 0x2 > 0
- 陷阱: type & 0x4 > 0
- 反击陷阱: type & TYPE_TRAP > 0 且 desc含"反击陷阱"或type含反击标志
- 效果怪兽: type & 0x20 > 0
- 调整怪兽: type & 0x1000 > 0

## setcode(系列)规则
- setcode是64位整数, 4×16位字段
- 提取方法: (setcode >> N*16) & 0xFFFF, N=0,1,2,3
- 常见系列: 烙印=1154, 相剑=1254, 自奏=1168, 深渊之兽=1234, 珠泪=1232
- 系列判定: 一张卡的4个16位字段中任意一个匹配目标系列=属于该系列
- 跨系列卡: 一张卡可能同时属于多个系列(如教导的惩罚同时属于教导和深渊之兽)

## 副卡组(Side)
- Side: 0-15张(可选)
- Side用于赛后换入换出, 不纳入本次构筑评估
- ydkcheck.py不检查Side合规性
